How to read this diagram

Solid items are implemented, tested, and dogfood-verified. Faded dashed items are planned but not yet built. Lifeblood is the hexagonal glue layer that normalizes language intelligence on the left and routes it into AI-facing surfaces on the right.

Adapters (left)Roslyn is the C# reference. TypeScript is the second proof. JSON graph is the universal bridge. Community builds Go, Python, Rust, Java.
Ports (center)Stable application seams. Left ports accept semantic intake. Right ports expose connector-ready truth. GraphDocument carries full metadata.
Connectors (right)Context packs, instruction files, MCP server with 6 interactive tools, CLI with CI exit codes. They depend inward, never on adapters.
This diagram is code-editable HTML/CSS/SVG. Solid = implemented. Dashed/faded = planned. The architecture is enforced by invariant tests and 11 frozen ADRs, not just convention. Self-analysis: dotnet run --project src/Lifeblood.CLI -- analyze --project . --rules packs/lifeblood/rules.json
