# Governance Invariants

Doc / repo discipline: pinned counts in `docs/STATUS.md`, complete
CHANGELOG link references, and test-pattern rules.

- **INV-DOCS-001. Doc numbers match the repository.** `docs/STATUS.md` declares port and tool counts in HTML comments (`<!-- portCount: N -->`, `<!-- toolCount: N -->`). The HTML comment is the single source of truth; `DocsTests` parses it and asserts the number matches the live count of `public interface I*` declarations under `src/Lifeblood.Application/Ports` (ports) and `Name = "lifeblood_*"` literals in `ToolRegistry.cs` (tools). Editing the count in one place and not the other fails the ratchet.

- **INV-DOCS-005. Invariant count anchor matches live audit.** `docs/STATUS.md` declares `<!-- invariantCount: N -->`. `DocsTests.StatusDoc_InvariantCount_MatchesLiveAudit` constructs a `LifebloodInvariantProvider` against the repo root, calls `Audit`, and asserts `audit.TotalCount` equals the anchor. Closes the drift class where invariant count cited in prose (README, ARCHITECTURE.md, architecture.html, devmemory) silently drifted from the live invariant tree because the prior `DocsTests` family had no invariant-count ratchet â€” sibling to `INV-DOCS-001` (portCount / toolCount), `INV-DOCS-004` (testCount).

- **INV-DOCS-006. Invariant category count anchor matches live audit.** `docs/STATUS.md` declares `<!-- invariantCategoryCount: N -->`. `DocsTests.StatusDoc_InvariantCategoryCount_MatchesLiveAudit` asserts `audit.CategoryCounts.Length` equals the anchor. Companion to `INV-DOCS-005`: a refactor that renames `INV-FOO-001` to `INV-FOO-NEW-001` may keep the same total count while changing the category set; the category-count ratchet catches that drift class.

- **INV-DOCS-007. `[SkippableFact]` count anchor matches live discovery.** `docs/STATUS.md` declares `<!-- skippableFactCount: N -->` alongside `<!-- testCount: N -->`. `DocsTests.StatusDoc_SkippableFactCount_MatchesLiveDiscovery` reflects on the test assembly and asserts the count of methods carrying `Xunit.SkippableFactAttribute` equals the anchor. The ratchet anchors the DECLARED `[SkippableFact]` surface, not the runtime-skip outcome â€” runtime skips depend on env gates (`LIFEBLOOD_REQUIRE_NATIVE_CLANG`, etc.) and cannot be mechanically checked without running the suite. Closes the drift class where prose surfaces (STATUS.md / README.md / architecture.html) shipped "zero skipped" copy while the test suite silently grew runtime-gated tests. Sibling to `INV-DOCS-001` (portCount / toolCount), `INV-DOCS-004` (testCount), `INV-DOCS-005`/006 (invariantCount / invariantCategoryCount).

- **INV-DOCS-SURFACE-PARITY-001. Every reviewer-facing public surface that cites the same count as a `STATUS.md` HTML-anchor MUST agree with the anchor.** Closes the trap class where `STATUS.md` anchors stayed current under their ratchets (`INV-DOCS-001` / `-004` / `-005` / `-006` / `-007`) while `README.md` / `docs/ARCHITECTURE.md` / `docs/architecture.html` silently kept stale prose ("27 ports", "29 Tools", "90 invariants") for multiple release cycles. The ratchet enumerates `[README.md, docs/ARCHITECTURE.md, docs/architecture.html]` as the surface-parity surfaces; for each, it scans the file with permissive phrasing patterns (`N tools` / `N MCP tools` / `N port[s] interfaces` / `N typed/queryable invariants`) and asserts every visible number equals the corresponding STATUS anchor. Plainly-different signals (small example counts, version strings) are excluded by an order-of-magnitude heuristic. Adding a new reviewer-facing surface that cites these counts requires extending the `SurfaceParitySurfaces` MemberData list â€” that act is the architecture decision. Pinned by `DocsTests.SurfaceParity_ToolCountAgreesWithStatusAnchor` + `SurfaceParity_PortCountAgreesWithStatusAnchor` + `SurfaceParity_InvariantCountAgreesWithStatusAnchor` (3 theories Ă— 3 surfaces = 9 cases). Sibling to `INV-LIST-SHAPE-UNIFORM-001` (eternal uniformity ratchet on a different axis).

- **INV-CHANGELOG-001. Every version heading has a link reference.** `CHANGELOG.md` must contain a `[X.Y.Z]: https://github.com/.../compare/...` link reference for every `## [X.Y.Z]` heading. Ratchet-tested by `DocsTests.Changelog_EveryHeadingHasLinkReference`. Closes the drift class where v0.6.0 shipped with stale bottom-of-file link refs.

- **INV-TESTDISC-001. Tests never silently early-return on precondition failure.** The `TryAnalyze(out ...) â‡’ bool` + `if (!TryAnalyze(...)) return;` pattern is forbidden â€” it hides both legitimate skips and real failures as silent passes. Missing preconditions (golden repo not restored) turn into `Skip.IfNot(condition, reason)` via `Xunit.SkippableFact`; broken-but-present conditions (graph has zero symbols) turn into loud `Assert.True` / `Assert.Fail`. Canonical example: `WriteSideIntegrationTests.cs`. Grep for `if (!Try*` under `tests/` must return zero hits.
