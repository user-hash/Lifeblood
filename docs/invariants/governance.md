# Governance Invariants

Doc / repo discipline: pinned counts in `docs/STATUS.md`, complete
CHANGELOG link references, and test-pattern rules.

- **INV-DOCS-001. Doc numbers match the repository.** `docs/STATUS.md` declares port and tool counts in HTML comments (`<!-- portCount: N -->`, `<!-- toolCount: N -->`). The HTML comment is the single source of truth; `DocsTests.Anchor_MatchesLiveSource` asserts the number matches the live count of `public interface I*` declarations under `src/Lifeblood.Application/Ports` (ports) and `Name = "lifeblood_*"` literals in `ToolRegistry.cs` (tools).

- **INV-DOCS-002. STATUS.md visible prose matches its own anchors.** Self-consistency check internal to STATUS.md. `DocsTests.StatusVisible_MatchesAnchor` iterates every anchor with a `VisiblePattern` and asserts each visible citation in STATUS.md equals its hidden anchor.

- **INV-DOCS-003. Visible read/write tool split matches `ToolRegistry` availability.** STATUS.md visible "(N read + M write)" / "(NR + MW)" prose must equal the `ToolAvailability.ReadSide` / `WriteSide` counts in `ToolRegistry.cs`. Two-number contract pinned by `DocsTests.StatusDoc_VisibleReadWriteSplit_MatchesRegistryAvailability`.

- **INV-DOCS-004. `testCount` anchor matches xUnit runtime expansion.** STATUS.md declares `<!-- testCount: N -->`. `DocsTests.Anchor_MatchesLiveSource` walks the test assembly via reflection, counts `[Fact]` + `[InlineData]` rows + `[MemberData]`/`[ClassData]` expansion of `[Theory]`, and asserts equality with the anchor.

- **INV-DOCS-005. Invariant count anchor matches live audit.** `docs/STATUS.md` declares `<!-- invariantCount: N -->`. `DocsTests.StatusDoc_InvariantCount_MatchesLiveAudit` constructs a `LifebloodInvariantProvider` against the repo root, calls `Audit`, and asserts `audit.TotalCount` equals the anchor. Closes the drift class where invariant count cited in prose (README, ARCHITECTURE.md, architecture.html, devmemory) silently drifted from the live invariant tree because the prior `DocsTests` family had no invariant-count ratchet — sibling to `INV-DOCS-001` (portCount / toolCount), `INV-DOCS-004` (testCount).

- **INV-DOCS-006. Invariant category count anchor matches live audit.** `docs/STATUS.md` declares `<!-- invariantCategoryCount: N -->`. `DocsTests.StatusDoc_InvariantCategoryCount_MatchesLiveAudit` asserts `audit.CategoryCounts.Length` equals the anchor. Companion to `INV-DOCS-005`: a refactor that renames `INV-FOO-001` to `INV-FOO-NEW-001` may keep the same total count while changing the category set; the category-count ratchet catches that drift class.

- **INV-DOCS-008. Self-analyze anchors match live `RoslynWorkspaceAnalyzer` output.** `docs/STATUS.md` declares `<!-- selfAnalyzeSymbols: N -->`, `<!-- selfAnalyzeEdges: N -->`, `<!-- selfAnalyzeModules: N -->`, `<!-- selfAnalyzeTypes: N -->`. `DocsTests.Anchor_MatchesLiveSource` runs the analyzer against the repo root via a lazy snapshot (one analyze per test session) and asserts each anchor equals its live counterpart.

- **INV-DOCS-009. Static-tables default anchors track `RoslynStaticTableExtractor` constants.** `docs/STATUS.md` declares `<!-- staticTablesDefaultMaxRows: N -->` and `<!-- staticTablesDefaultMaxTables: N -->`. `DocsTests.Anchor_MatchesLiveSource` reads `RoslynStaticTableExtractor.DefaultMaxRows` / `DefaultMaxTables` (internal `const int`, visible via `InternalsVisibleTo` to `Lifeblood.Tests`) and asserts equality. The constant is the SoT; the anchor mirrors it; prose surfaces reference STATUS.md instead of carrying literal numbers.

- **INV-DOCS-007. `[SkippableFact]` count anchor matches live discovery.** `docs/STATUS.md` declares `<!-- skippableFactCount: N -->` alongside `<!-- testCount: N -->`. `DocsTests.StatusDoc_SkippableFactCount_MatchesLiveDiscovery` reflects on the test assembly and asserts the count of methods carrying `Xunit.SkippableFactAttribute` equals the anchor. The ratchet anchors the DECLARED `[SkippableFact]` surface, not the runtime-skip outcome — runtime skips depend on env gates (`LIFEBLOOD_REQUIRE_NATIVE_CLANG`, etc.) and cannot be mechanically checked without running the suite. Closes the drift class where prose surfaces (STATUS.md / README.md / architecture.html) shipped "zero skipped" copy while the test suite silently grew runtime-gated tests. Sibling to `INV-DOCS-001` (portCount / toolCount), `INV-DOCS-004` (testCount), `INV-DOCS-005`/006 (invariantCount / invariantCategoryCount).

- **INV-DOCS-SURFACE-PARITY-001. Every reviewer-facing surface that cites a STATUS.md HTML-anchor count MUST agree with the anchor; the preferred eternal shape is to strip the duplicate citation and point to STATUS.md.** SSoT is the `DocsAnchor[]` table in `DocsTests.cs`: each entry carries name + live-source delegate + optional visible pattern. Three Theories iterate the table — `Anchor_MatchesLiveSource` (anchor↔live), `StatusVisible_MatchesAnchor` (STATUS.md's own visible prose↔its own anchor), `NonStatusSurface_VisibleCitationMatchesAnchor` (every non-STATUS surface↔anchor). Surfaces that strip the duplication match nothing and pass silently. Adding a ratcheted count = adding one table row. Sibling to `INV-LIST-SHAPE-UNIFORM-001`.

- **INV-TRACKING-SSOT-001. The living implementation ledger is `devmemory/lifeblood-tracking.md`, and its summary / active-backlog surface MUST be generated from the entry bodies by test, not hand-maintained prose.** `TrackingLedgerTests` parses every real status-bearing ledger section, classifies its `Status:` line, asserts the hidden status-count anchors match the parsed body count, asserts the active-backlog block equals the `Partially shipped` entries, and requires every partially shipped entry to name `Remaining open work:`. The tracker may carry narrative context, but closure evidence stays in the entry body before the summary can move.

- **INV-INTAKE-SHAPE-001. The intake file `devmemory/lifeblood-intake.md` is the provenance-critical front door for un-started findings, and its entry shape MUST be ratcheted even though it carries no status model.** `IntakeLedgerTests` parses every level-2 heading that begins `LB-INTAKE-`, asserts each id matches `^LB-INTAKE-\d{8}-\d{3}$`, asserts ids are unique, requires `Type:` / `Priority:` / `Source:` / `Workspace:` metadata and the `What:` / `Why it matters:` / `Fix shape:` authoring sections on every entry, flags any heading that mentions `LB-INTAKE` but is not a canonical id, and forbids an intake id from also appearing in `devmemory/lifeblood-tracking.md` (promotion must move, not copy). Sibling to `INV-TRACKING-SSOT-001`; deliberately shape/metadata-only because intake is un-prioritized until promoted.

- **INV-CHANGELOG-001. Every version heading has a link reference.** `CHANGELOG.md` must contain a `[X.Y.Z]: https://github.com/.../compare/...` link reference for every `## [X.Y.Z]` heading. Ratchet-tested by `DocsTests.Changelog_EveryHeadingHasLinkReference`. Closes the drift class where v0.6.0 shipped with stale bottom-of-file link refs.

- **INV-TESTDISC-001. Tests never silently early-return on precondition failure.** The `TryAnalyze(out ...) ⇒ bool` + `if (!TryAnalyze(...)) return;` pattern is forbidden — it hides both legitimate skips and real failures as silent passes. Missing preconditions (golden repo not restored) turn into `Skip.IfNot(condition, reason)` via `Xunit.SkippableFact`; broken-but-present conditions (graph has zero symbols) turn into loud `Assert.True` / `Assert.Fail`. Canonical example: `WriteSideIntegrationTests.cs`. Grep for `if (!Try*` under `tests/` must return zero hits.
